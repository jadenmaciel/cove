<objective>
Implement core workspace save/restore: new workspaces get PermanentWorkspaceRecord entries in WorkspaceRegistryStore, layout snapshots saved via SessionPersistenceStore, and both restore on app launch via TabManager.
</objective>

<context>
@Sources/WorkspaceRegistry.swift
@Sources/SessionPersistence.swift
@Sources/Workspace.swift
@Sources/TabManager.swift

## Locked Decisions Implemented

- **D-01:** Auto-name from directory ‚Äî new workspaces default to root directory name via `liveProjectWorkspaceName()`
- **D-06:** Layout-only restore ‚Äî `includeScrollback: false` in sessionSnapshot()
- **D-08:** Manual creation only ‚Äî Cmd+N creates new workspace with registry record

## Architecture

```
User Action (Cmd+N or sidebar button)
        |
        v
TabManager.addTab() --> Creates Workspace + PermanentWorkspaceRecord
        |                           |
        | (sessionSnapshot)          | (WorkspaceRegistryStore.save)
        v                           v
SessionPersistenceStore         ~/Library/Application Support/cove/
(saves layout-{id}.json)        workspaces-{bundleID}.json
```

## Layout File Convention

Per-workspace layout files stored at:
`~/Library/Application Support/cove/sessions/workspaces/{workspaceId}/layout.json`

This keeps per-workspace layouts isolated from the single-session snapshot.
</context>

<task type="auto">
  <name>Task 1: Wire Workspace creation to WorkspaceRegistry</name>
  <files>Sources/TabManager.swift</files>
  <action>
    Modify `TabManager.addTab()` to create a `PermanentWorkspaceRecord` when creating a new workspace.

    1. In `addTab()`, after workspace is created, call `WorkspaceRegistryStore.load()` to get existing registry
    2. Create new `PermanentWorkspaceRecord` with:
       - `id`: `workspace.id` (preserve the workspace's UUID)
       - `kind`: `.project`
       - `name`: `liveProjectWorkspaceName(for: workspace)` ‚Äî derives from directory per D-01
       - `rootPath`: `workspace.currentDirectory`
       - `pinned`: `false`
       - `sortIndex`: next available index
       - `restorePolicy`: `.restoreLastSession`
       - `createdAt`: `Date()`
       - `updatedAt`: `Date()`
    3. Append record to `registry.workspaces`
    4. Call `WorkspaceRegistryStore.save(registry, fileURL: registryURL)`

    The layout file URL convention is `sessions/workspaces/{workspaceId}/layout.json` within the App Support directory.
  </action>
  <verify>
    <automated>grep -n "PermanentWorkspaceRecord" Sources/TabManager.swift | head -20</automated>
  </verify>
  <done>PermanentWorkspaceRecord created and saved when new workspace added via addTab()</done>
</task>

<task type="auto">
  <name>Task 2: Capture layout snapshot on workspace creation</name>
  <files>Sources/TabManager.swift, Sources/SessionPersistence.swift</files>
  <action>
    After creating PermanentWorkspaceRecord in addTab(), capture the initial layout snapshot.

    1. Call `workspace.sessionSnapshot(includeScrollback: false)` on main actor (D-06: no scrollback)
    2. Save snapshot to layout file URL: `sessions/workspaces/{workspaceId}/layout.json`
    3. Use `SessionPersistenceStore.save(snapshot, fileURL: layoutURL)` ‚Äî nonisolated, safe to call off-main

    Note: Use `await MainActor.run { workspace.sessionSnapshot(includeScrollback: false) }` since sessionSnapshot is @MainActor but save is nonisolated.
  </action>
  <verify>
    <automated>grep -n "sessionSnapshot.*includeScrollback" Sources/TabManager.swift</automated>
  </verify>
  <done>Initial layout snapshot captured and saved for new workspace</done>
</task>

<task type="auto">
  <name>Task 3: Restore workspaces on app launch</name>
  <files>Sources/TabManager.swift, Sources/WorkspaceRegistry.swift, Sources/SessionPersistence.swift</files>
  <action>
    On app launch, restore permanent workspaces from registry with their layouts.

    1. In TabManager init or AppDelegate restore flow, load registry: `WorkspaceRegistryStore.load(fileURL: registryURL)`
    2. Filter to non-archived workspaces: `registry.workspaces.filter { $0.status != .archived }`
    3. Sort by pinned first, then sortIndex, then updatedAt (existing pattern)
    4. For each record:
       a. Call `addTab(title: record.name, workingDirectory: record.rootPath)` to create workspace
       b. Preserve `workspace.id = record.id` after creation
       c. Restore pinned state: `workspace.isPinned = record.pinned`
       d. Load layout from `sessions/workspaces/{record.id}/layout.json` via `SessionPersistenceStore.load(fileURL: layoutURL)`
       e. If layout exists, call `workspace.restoreSessionSnapshot(layout)` (D-06: layout only, no scrollback)
    5. Select first workspace or last active workspace per restore policy

    IMPORTANT: Use `Workspace.restoreSessionSnapshot()` directly ‚Äî do not bypass with custom restore logic.
  </action>
  <verify>
    <automated>grep -n "restoreSessionSnapshot" Sources/TabManager.swift</automated>
  </verify>
  <done>All permanent workspaces restored with correct layouts on app launch</done>
</task>
